# Contributing to AstraTrip

First off, thank you for considering contributing to AstraTrip! 

## Branching Strategy
- `main`: Protected. Represents the current production state.
- `feature/<name>`: Used for developing new features.
- `fix/<name>`: Used for bug fixes.

## Pull Request Process
1. Ensure any install or build dependencies are removed before the end of the layer when doing a build.
2. Update the README.md with details of changes to the interface, this includes new environment variables, exposed ports, useful file locations and container parameters.
3. You may merge the Pull Request in once you have the sign-off of at least one other developer.
